## 1Ô∏è‚É£ Basic @Preview üîç

### Konsep Dasar
`@Preview` seperti jendela ajaib üîÆ - menunjukkan tampilan UI tanpa harus menjalankan aplikasi!

### Implementation
**ui/screens/PreviewExamplesScreen.kt**
```kotlin
package com.example.mycomposeapp.ui.screens

import androidx.compose.foundation.background
import androidx.compose.foundation.layout.*
import androidx.compose.foundation.shape.RoundedCornerShape
import androidx.compose.material.icons.Icons
import androidx.compose.material.icons.filled.*

import androidx.compose.ui.unit.dp
import androidx.compose.ui.unit.sp

// Contoh basic composable
@Composable
fun GreetingCard(
    name: String,
    onClick: () -> Unit = {}
) {
    Card(
        modifier = Modifier
            .fillMaxWidth()
            .padding(16.dp),
        shape = RoundedCornerShape(12.dp),
        elevation = CardDefaults.cardElevation(
            defaultElevation = 8.dp
        )
    ) {
        Row(
            modifier = Modifier
                .fillMaxWidth()
                .padding(16.dp),
            verticalAlignment = Alignment.CenterVertically,
            horizontalArrangement = Arrangement.spacedBy(12.dp)
        ) {
            Icon(
                imageVector = Icons.Default.Person,
                contentDescription = "User",
                tint = MaterialTheme.colorScheme.primary
            )
            Column(
                modifier = Modifier.weight(1f)
            ) {
                Text(
                    text = "Halo, $name!",
                    fontWeight = FontWeight.Bold,
                    fontSize = 18.sp
                )
                Text(
                    text = "Selamat datang di aplikasi kami",
                    fontSize = 14.sp,
                    color = MaterialTheme.colorScheme.onSurfaceVariant
                )
            }
            IconButton(
                onClick = onClick
            ) {
                Icon(
                    imageVector = Icons.Default.Favorite,
                    contentDescription = "Favorite"
                )
            }
        }
    }
}

    TitleText("Preview Title")
```

### üéì Penjelasan Konsep:
**@Preview seperti Jendela Ajaib:**
- `@Preview` annotation memberi tahu Compose untuk menampilkan preview
- Tidak perlu menjalankan aplikasi untuk melihat tampilan
- Otomatis direfresh saat kode berubah
- Bisa digunakan untuk semua @Composable function

**Cara Menggunakan:**
1. Tambahkan `@Preview` annotation di atas composable function
2. Pastikan composable function memiliki nama unik untuk preview
3. Tambahkan `showBackground = true` untuk background putih

```kotlin
                    ) {
                        Icon(Icons.Default.Error, contentDescription = "Error", tint = Color.Red)
                        Text("Terjadi kesalahan", color = Color.Red)
                        Button(
                            onClick = { /* Handle retry */ },
                            modifier = Modifier.padding(top = 8.dp)
                        ) {
                            Text("Coba Lagi")
                        }
                    }
                }
            }
        }
        DataState.Success -> {
            LazyColumn {
                items(data.size) { index ->
                    Text(
                        text = data[index],
                        modifier = Modifier
                            .fillMaxWidth()
                            .padding(16.dp)
                    )
                }
            }
        }
    }
}
```

- Jangan menggantungkan pada data eksternal
- Buat data sample yang representative

**5. Test responsiveness:**
- Gunakan berbagai ukuran layar
- Testing orientasi berbeda
- Pastikan UI tampil baik di semua ukuran

- Light dan dark theme
- Berbagai konfigurasi sistem

## üéØ Kesimpulan

Dengan menguasai Compose Previews, kamu sekarang memiliki kemampuan untuk:

- **Menggunakan Basic @Preview**: Membuat preview sederhana untuk UI
- **Menggunakan Preview Parameters**: Mengatur berbagai konfigurasi preview
- **Membuat Multiple Previews**: Menampilkan berbagai kondisi UI
- **Menggunakan @PreviewParameter**: Membuat banyak preview dengan data berbeda
- **Membuat Preview Groups**: Mengelompokkan preview dengan tema sama
- **Menggunakan Interactive Preview Mode**: Menguji interaksi langsung dalam preview
- **Menerapkan Best Practices**: Mengikuti panduan untuk preview yang efektif

### üî• Tips Penting:
1. Gunakan berbagai `@Preview` untuk kondisi berbeda (loading, empty, error, success)
2. Implementasikan preview responsive untuk berbagai ukuran layar
3. Gunakan `@PreviewParameter` untuk menguji banyak data secara otomatis
4. Buat custom preview groups untuk konfigurasi yang sering digunakan
5. Gunakan preview untuk menguji tema (light/dark) dan konfigurasi sistem
6. Jaga preview functions tetap ringan dan tidak tergantung pada data eksternal
7. Gunakan data sample khusus untuk preview, bukan data production
8. Test edge cases dalam preview (data kosong, terlalu panjang, dll)
9. Gunakan interactive preview untuk uji coba cepat interaksi UI
10. Gunakan preview sebagai dokumentasi visual untuk komponen UI

Kamu sekarang siap membuat preview yang membantu dalam pengembangan UI yang efisien dan konsisten!
