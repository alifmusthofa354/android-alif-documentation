#### 3. HTTP Methods (CRUD Operations)
**GET:**
- Mengambil data dari server
- Idempotent operation
- Query parameters
- Example: `GET /api/users/123`

**POST:**
- Create new resource
- Non-idempotent
- Request body required
- Example: `POST /api/users` with JSON body

**PUT:**
- Update entire resource
- Idempotent
- Replace existing resource
- Example: `PUT /api/users/123`

**PATCH:**
- Partial update resource
- Update specific fields only
- Example: `PATCH /api/users/123`

**DELETE:**
- Remove resource
- Idempotent
- Example: `DELETE /api/users/123`

#### 4. REST Principles & Constraints
**1. Client-Server Separation:**
- Independent evolution
- Separation of concerns

**2. Stateless:**
- Setiap request independent
- No session state di server
- All context dalam request

**3. Cacheable:**
- Responses harus define cacheable atau tidak
- Performance optimization

**4. Uniform Interface:**
- Resource identification (URI)
- Resource manipulation through representations
- Self-descriptive messages
- HATEOAS (optional)

**5. Layered System:**
- Client tidak tahu intermediate layers
- Load balancers, proxies, gateways

**6. Code on Demand (Optional):**
- Server can send executable code

#### 5. JSON (JavaScript Object Notation)
**JSON Basics:**
```json
{
  "id": 1,
  "name": "John Doe",
  "email": "john@example.com",
  "age": 30,
  "active": true,
  "address": {
    "street": "123 Main St",

```

**JSON Data Types:**
- String: "text"
- Number: 123, 45.67
- Boolean: true, false
- Null: null
- Object: { }

**JSON vs XML:**
- Lebih lightweight
- Easier to parse
- Native JavaScript support
- Human-readable

#### 6. HTTP Status Codes
**2xx Success:**
- 200 OK - Request succeeded
- 201 Created - Resource created
- 204 No Content - Success, no body

**3xx Redirection:**
- 301 Moved Permanently
- 302 Found (temporary redirect)
- 304 Not Modified (cache)

**4xx Client Errors:**
- 400 Bad Request - Invalid syntax
- 401 Unauthorized - Authentication required
- 403 Forbidden - No permission
- 404 Not Found - Resource doesn't exist
- 422 Unprocessable Entity - Validation error

**5xx Server Errors:**
- 500 Internal Server Error
- 502 Bad Gateway
- 503 Service Unavailable

#### 7. RESTful URL Design
**Best Practices:**
```
✅ GOOD:
GET    /api/users           - Get all users
GET    /api/users/123       - Get user by ID
POST   /api/users           - Create user
PUT    /api/users/123       - Update user
DELETE /api/users/123       - Delete user
GET    /api/users/123/posts - Get user's posts

❌ BAD:
GET /api/getUsers
POST /api/createUser
GET /api/user_123
```

**Conventions:**
- Use nouns, not verbs
- Plural names for collections
- Hierarchical structure
- Lowercase letters
- Hyphens for multi-word (optional)
- Version dalam URL: `/api/v1/users`

#### 8. API Endpoints & Resources
- Resource identification dengan URI
- Collection vs single resource
- Nested resources
- Filtering: `/api/users?role=admin`
- Pagination: `/api/users?page=2&limit=20`
- Sorting: `/api/users?sort=name&order=asc`
- Search: `/api/users?q=john`

#### 9. Request & Response Format
**Request Example:**
```http
POST /api/users HTTP/1.1
Host: api.example.com
Content-Type: application/json
Authorization: Bearer token123

{
  "name": "John Doe",
  "email": "john@example.com"
}
```

**Response Example:**
```http
HTTP/1.1 201 Created
Content-Type: application/json

{
  "id": 123,
  "name": "John Doe",
  "email": "john@example.com",
  "created_at": "2024-01-15T10:30:00Z"
}
```
