# Use Cases

## Deskripsi
Use cases (juga dikenal sebagai Interactors) digunakan untuk memisahkan business logic dari ViewModel. Anda akan belajar bagaimana mengimplementasikan domain layer.

## Tujuan Pembelajaran
- Memahami konsep Use Cases
- Memisahkan business logic dari ViewModel
- Menggunakan Use Cases dalam arsitektur clean

## Materi
- Konsep Use Cases/Interactors
- Implementasi Use Cases
- Domain layer dalam Clean Architecture
- Dependency inversion dalam Use Cases
- Penggantian logic in ViewModel
- Kapan menggunakan Use Cases

## Waktu
3 hari

## Prioritas
⭐⭐ (Optional tapi recommended)

## Catatan
Use Cases adalah opsional tapi direkomendasikan untuk aplikasi dengan business logic kompleks.
